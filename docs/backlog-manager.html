Backlog Manager

A generic subsystem for managing backlog files that coordinate multi-agent work.

Vision

Backlog Manager exists because LLM agents perform better when tasks are written down with explicit status, just as human teams do. By keeping a durable backlog of tasks, the system reduces hallucinations, improves continuity, and makes progress visible.

The manager is generic by design. It supports any number of backlog types so teams can maintain separate backlogs for specifications, documentation, operations, or any other domain that benefits from structured task tracking.

What It Does

Backlog Manager provides APIs for reading, updating, and organizing backlog files. It treats each backlog as a structured list of tasks with statuses, enabling agents to coordinate through a shared source of truth rather than ad‑hoc memory.

Usage

Use Backlog Manager whenever you need persistent task coordination across skills or sessions. It is especially effective in workflows where tasks are discovered over time and need to be tracked across multiple agent runs.
