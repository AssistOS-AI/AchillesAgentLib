MCP Skills

Skills that integrate Model Context Protocol servers as an execution backend.

Vision

MCP skills exist to connect the skill system to external MCP servers. When a server exposes tools that can fulfill a user request, an MCP skill can plan a sequence of calls and optionally execute them. This keeps external integrations inside the same skill‑driven model rather than as ad‑hoc tool calls.

The flow is lighter than a full agentic session. It generates a compact plan and executes it with minimal overhead, while still using the same skill allowlist and safety boundaries.

How It Works

An MCP skill can operate in two modes. If a LightSOPLang script is present, the script builds the plan. Otherwise the subsystem asks the LLM to generate a plan from the allowed tools. In both cases the result is a list of tool calls with arguments.

Execution is optional. If an MCP client is configured, the subsystem will call the MCP server for each step. If no client is provided, it returns the plan only.

Usage

MCP skills are used when you want to route a request through a server‑side MCP toolchain. They are useful for connecting to external systems that already expose MCP tools without rewriting them as internal skills.

Folder Structure

MCP skills live in their own folder and are defined by a single mskill.md descriptor file. 

skills/
└── mcp-file-manager/
    └── mskill.md

mskill.md Sections

MCP skill descriptors control how plans are generated and which MCP tools are allowed.

  • Instructions provides guidance for planning or scripting.
  • Allowed-Tools restricts the MCP tools that can be used.
  • Light-SOP-Lang optionally provides a script to build the plan directly.
