Dynamic Code Generation Skills

Adaptive skills that generate and execute code on the fly for precise, context‑aware work.

Vision

Dynamic code generation skills exist for situations where a precise computation is required and a static skill would be too rigid. This is the same motivation behind LLMs that run a temporary script to compute an exact result. The skill remains intelligent and context‑aware, but it can produce code when code is the safest way to reach the answer.

The descriptor constrains what the skill should do, while still leaving room to adapt based on context. That “wiggle room” is essential for tasks whose exact computation depends on the situation at runtime.

How It Works

The subsystem asks the model whether to respond directly or to generate code. When code is generated, it is executed immediately using eval and the result is returned. Nothing is persisted to disk; the generated code is temporary and discarded after execution.

This is different from Code Skills, where code is generated from specifications and stored under src/. Dynamic code generation is intentionally ephemeral.

The primary mode is dynamic generation. If a skill already ships with a module on disk, the subsystem will execute that module instead, which saves time for stable cases while keeping the dynamic option available when needed.

Folder Structure

A dynamic code generation skill can be descriptor‑only, or it can include a module that will be executed directly. 

skills/
└── math-evaluator/
    ├── dcgskill.md
    └── math-evaluator.mjs

dcgskill.md Sections

The descriptor is minimal but explicit. It defines the prompt, the input argument, and how the model should behave.

  • Prompt explains the decision logic and the intended behavior.
  • Argument defines the primary input label.
  • LLM-Mode sets whether the skill runs in fast or code mode.
