Internal Skills

Internal skills are built-in code skills (cskill) that provide core functionality for file operations, shell commands, web fetching, and user interaction. They are automatically discovered from the skills/ directory in the package root and are available to all subsystems.

Overview

Internal skills are implemented as code skills with a cskill.md descriptor and a src/index.mjs module that exports an action(context) function. They are marked as isInternal: true during registration and can be optionally hidden from LLM skill selection using the exposeInternalSkills flag.

These skills are used by AnthropicSkillsSubsystem to build tools for agentic loop sessions, and can be invoked directly by orchestrators or other skills.

Available Internal Skills

ask-user

Prompts the user for missing or unclear information and returns the response as plain text.

Input: A question or clarification text (string).

Output: User's response (string).

Example:

What is your target audience?

read

Reads file contents from disk. Returns numbered lines for text files and base64 for binary files.

Input: Key-value pairs with file_path (required), offset (optional), limit (optional).

Output: Numbered lines or base64 string.

Example:

file_path: /abs/path/file.txt
offset: 101
limit: 50

write

Creates or overwrites a file with new content. Creates parent directories if needed.

Input: Key-value pairs with file_path (required), content (required).

Output: Success message with character count.

Example:

file_path: /abs/path/file.txt
content: hello world

edit

Replaces exact text in a file. Supports single or multiple replacements.

Input: Key-value pairs with file_path, old_string, new_string, replace_all (optional).

Output: Success message.

Example:

file_path: /abs/path/app.js
old_string: debug=true
new_string: debug=false
replace_all: true

glob

Finds files by glob pattern. Returns a JSON array of absolute file paths sorted by modification time (most recent first).

Input: Key-value pairs with pattern (required), path (optional base directory).

Output: JSON array of file paths.

Example:

pattern: **/*.js
path: src

grep

Searches file contents using regex patterns. Supports multiple output modes and context lines.

Input: Key-value pairs with pattern (required), path, glob, output_mode (files_with_matches, content, count), -i, -n, -C, etc.

Output: Depends on output_mode - file paths, matching lines, or counts.

Example:

pattern: TODO
path: /abs/project
output_mode: files_with_matches

bash

Runs a shell command via bash in the current working directory.

Input: Key-value pairs with command (required), timeout (optional).

Output: stdout, stderr (if present), exit code (if non-zero), timeout flag (if timed out).

Example:

command: ls -la
timeout: 60000

webfetch

Fetches web content from a URL. HTML is automatically converted to plain text.

Input: Key-value pairs with url (required), prompt (required description).

Output: Plain text content.

Example:

url: https://example.com
prompt: summarize

mirror-code-generator

Generates JavaScript/ESM code from a skill's specs/ directory. Reads .md/.mds spec files, produces or updates matching src/ files, and backs up specs to specs/.backup after a successful run.

Input: Absolute or relative path to the target skill directory containing a specs/ folder.

Output: Object with message and generatedFiles array.

Example:

./skills/inventory

Visibility Control

Internal skills can be hidden from LLM skill selection using the exposeInternalSkills flag in RecursiveSkilledAgent: 

const agent = new RecursiveSkilledAgent({
    llmAgent: myLLMAgent,
    exposeInternalSkills: false, // Hide from LLM selection
});

// Internal skills remain callable directly
await agent.executeWithReviewMode(
    'file_path: test.txt',
    { skillName: 'read' },
    'none'
);

When exposeInternalSkills is false, internal skills are filtered out in chooseSkillWithLLM() but remain fully functional when invoked explicitly by name.

Usage in Subsystems

Internal skills are automatically converted to tools by AnthropicSkillsSubsystem using buildAnthropicTools(). Each internal skill becomes a tool with its descriptor content as the description and a handler that delegates to recursiveAgent.executePrompt().

Orchestrator skills can also invoke internal skills directly through the standard skill execution flow.

Adding New Internal Skills

To add a new internal skill:

  1. Create a directory in skills/<skill-name>/
  2. Add a cskill.md descriptor with Description, Input Format, and Output Format sections
  3. Create src/index.mjs with an exported action(context) function
  4. The skill will be automatically discovered and registered as internal
// src/index.mjs
