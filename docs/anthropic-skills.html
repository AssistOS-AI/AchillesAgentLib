Anthropic Skills

Integration of the Anthropic skill standard for structured, reusable guidance.

Vision

Anthropic skills exist to integrate a widely used standard into this system. The standard provides a predictable structure for describing skills, and that structure is valuable even when the skill itself does not execute logic. By adopting it, we make cross‑team collaboration easier and keep skill definitions portable.

For reference, the standard is described here: Anthropic Skill Overview.

How They Behave Here

In this library, Anthropic skills are treated like orchestration skills that do not execute directly. They contribute structured guidance and resources to an agentic session, but the execution happens elsewhere through other skills. This keeps them aligned with the system’s “skills, not tools” model while preserving the intent of the Anthropic format.

At execution time, the system starts a Loop agentic session and uses it to run built-in skills that match the tools described in SKILL.md. The Anthropic skill itself does not execute; it guides the selection and invocation of those built-in skills.

Unlike a standard orchestration skill, an Anthropic skill can optionally include supporting folders for scripts, resources, and assets. These add context or supporting logic without breaking the static nature of the descriptor itself.

Folder Structure

An Anthropic skill lives in its own directory and can include optional folders used by the surrounding system. 

skills/
└── my-anthropic-skill/
    ├── skill.md
    ├── scripts/
    ├── resources/
    └── assets/

scripts/ can contain executable helpers, resources/ holds files that should be injected into agentic session context, and assets/ stores supporting files that are used indirectly by code but are not injected into the context directly.

skill.md Sections

The descriptor follows the Anthropic skill format. At minimum it includes a title and summary, and can include detailed body content that provides guidance, constraints, or examples.

  • Title (the first heading) names the skill.
  • Summary provides a short statement of purpose.
  • Body content describes the skill in narrative form.
