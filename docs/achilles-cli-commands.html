Achilles CLI — Commands and Prompt Handling

A deeper look at how the CLI reads prompts, stores history, and exposes runtime commands for humans and agents.

Prompt Editing & History

  • History file — every prompt (including slash commands) is appended to .specs/.prompts_history. Subsequent sessions preload the same file so arrow keys immediately recall earlier prompts.
  • Multi-line input — end a line with \ to continue typing under the ... prompt. Blank lines terminate multi-line mode.
  • Special keys — ↑/↓ browse history; Esc cancels the active plan (see below); Ctrl+C cancels the plan or exits if idle.
  • Non-interactive mode — pass --non-interactive when embedding the CLI so prompts come from your own reader; in that mode the internal history manager is bypassed.

Runtime Commands

Command Details
/help Prints command usage, history shortcuts, and cancellation hints.
/list Lists every available skill (name, type, summary, implementation hints). Honors --skills / ACHILLES_CLI_SKILLS overrides.
/debug [on|off] Toggles verbose LLM_DEBUG: logging. The same mode can be pre-enabled via ACHILLES_LLM_DEBUG=1.
/continue Resume the most recent plan. If you provide extra text (/continue incorporate telemetry) the CLI replans before resuming.
/resume Alias of /continue.
/run <skill> <<prompt>> Invoke a specific skill without running the planner. Example: /run spec-review <<Audit DS-003 for gaps>>.
/status Shows request counts, token totals, per-model stats, and the paths to .llm_logs/.llm_stats.
/cancel Cancel the active plan. Equivalent to pressing Esc or Ctrl+C while a plan is running.
/exit, /quit Gracefully close the CLI. Pending history and memories are flushed to disk.

Language Contract & /lang

All specification artifacts (URS/FS/NFS/DS, file-impact tables, and confirmation prompts) are emitted in English by default. The CLI injects an explicit “Language Requirements” block into every planner call and skill execution, preventing LLMs from drifting into other languages even when the operator submits prompts in Romanian or another locale.

  • Default — English output for every spec. The reminder is prepended to the planner prompt and every skill invocation.
  • /lang <code> — sets the desired specification language (for example /lang english, /lang romanian). The CLI normalizes the value and persists it for the rest of the session.
  • /lang (without arguments) — prints the current specification language so you can confirm the setting before issuing a new request.
  • UI localization vs specs — even if the UI echoes prompts in another language, the CLI forces spec text to the configured language and rejects placeholder strings such as your_value.

Cancellation & Resume

The CLI maintains a pending plan state. Every time a step starts, its index is recorded. Cancelling via Esc, Ctrl+C, or /cancel sets the plan to paused and prints how many steps remain:

[info] Plan paused with 3 step(s) remaining. Use \"continue\" or \"/continue\" to resume.

You can then:

  • Resume untouched by simply typing continue (any language variants such as “continúa” are detected) or using /continue.
  • Resume with tweaks (continue justifică FS-004) — the CLI replans using the previous prompt plus the extra instruction.
  • Abort entirely by entering a brand-new prompt (the pending plan is replaced).

Environment & Flags

  • ACHILLES_CLI_SKILLS — comma-separated list of directories that contain .AchillesSkills. Matches the --skills flag.
  • ACHILLES_LLM_DEBUG — set to 1/true to start the CLI with /debug on.
  • --bootstrap-mode — choose auto (default), ask, or manual. Remember that bootstrapping only runs when .specs/ is missing.
  • --assume-yes / --confirm-plan — explicitly skip or require plan confirmations.
  • --non-interactive — disable the readline wrapper (suitable for programmatic use).

LLM Status & Logs

The CLI captures every prompt/response pair under the workspace’s .specs directory:

  • .specs/.llm_logs — newline-delimited JSON entries containing prompt/response previews, token estimates, duration, and the model name used for each call.
  • .specs/.llm_stats — aggregate counters used by /status. The CLI shows total requests, per-model volume, and duration-based response buckets (<100, <1000, <10000, <100000, >100000 milliseconds).
  • /status — reports request counts, the last-used model, bucket statistics (only populated when a bucket has at least one request), and the log/stat file paths so you can tail them while debugging.

Because the log files live in the workspace, every project keeps its own audit trail without polluting the global home directory.
