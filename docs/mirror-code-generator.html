Mirror Code Generator

The mirror code generator is the shared, internal utility that transforms specs/ markdown files into executable modules for code skills. It performs regeneration only when specs are newer than their target outputs and backs up specs after successful generation.

Internal Skill: mirror-code-generator

The mirror-code-generator is the orchestration layer that turns spec markdown into runnable code, validates the output, and repairs failures. It evaluates each spec file individually and only regenerates code when the spec is newer than its target output (or the output is missing). Each spec path maps to a target file under src/ by removing the specs/ prefix and the .md/.mds extension. 

specs/index.mjs.mds  ->  src/index.mjs
specs/utils/foo.mjs.md  ->  src/utils/foo.mjs

For every stale spec, the LLM receives three inputs: the current spec content, the previous spec from specs/.backup (when present), and the existing code file (when present). The generator writes the new code to disk, runs node --check, and triggers a repair pass on syntax failure. If a second syntax check fails, the previous on-disk version is restored (or the file is removed).

Tests and DS-to-FDS generation are now handled externally. Mirror-code-generator focuses strictly on translating existing specs/ files into src/ outputs and writing specs/.backup after a successful regeneration pass.

Generation Flow (ASCII)

Start
  |
  v
Check specs/ exists?
  |-- No --> Stop (no specs)
  |
  v
Check if any spec is newer than its code?
  |-- No --> Stop
  |
  v
For each spec in specs/ (excluding .backup)
  |
  v
Read spec -> Gather spec + backup + existing code
  |
  v
Generate code -> Write to src/
  |
  v
Syntax check (node --check)
   |-- Fail --> Repair (spec + backup + generated code + syntax error)
   |           -> Rewrite -> Recheck
   |           -> Fail? Revert
  |-- Pass --> Continue
  |
  v
Overwrite specs/.backup
  |
  v
Done
