SpecsManager

SpecsManager is the shared utility for parsing and manipulating DS and FDS markdown files. It extracts section content, updates sections in-place, and provides helpers for dependency and export lists. This keeps spec handling consistent across generators and evaluators.

Role in the System

SpecsManager provides a single source of truth for interpreting spec structure. It normalizes how headings are parsed into sections, how dependency entries are read, and how updates are written back to files without losing the rest of the document. This enables tooling such as mirror-code-generator to enrich dependencies and safely edit specs.

DS Structure

A DS (Design Spec) captures high-level intent and scope. SpecsManager expects these sections: 

# Vision and Problem Statement
# Intended Users and Context of Use
# Scope and Boundaries
# Success Criteria
# Affected Files

The Affected Files section lists the FDS files that should be produced or updated. 

# Affected Files
- specs/index.mjs.md - Core module
- specs/utils/formatter.mjs.md - Formatting helpers

FDS Structure

An FDS (File Design Spec) describes a single output module. SpecsManager recognizes these sections: 

# Description
# Dependencies
# Main Functions
# Exports
# Implementation Details
# Testing

Dependencies are recorded as bullet entries in the format: 

- src/utils/math.mjs - sumValues : Used to total inputs

FDS Example

# Description
Provides a minimal counter utility.

# Dependencies
- src/utils/math.mjs - clamp : Ensures limits are enforced

# Main Functions
- createCounter(initial, options)
  - returns an object with increment, decrement, getValue

# Exports
- createCounter

# Implementation Details
Use closures to store internal state and enforce min/max limits.

# Testing
- should increment from initial value
- should clamp at max limit

DS Example

# Vision and Problem Statement
We need a reusable counter utility for small modules.

# Intended Users and Context of Use
Developers working on CLI and tooling modules.

# Scope and Boundaries
In-memory counters only. No persistence.

# Success Criteria
- Counter updates are predictable
- Limits are enforced

# Affected Files
- specs/counter/index.mjs.md - Counter module spec
