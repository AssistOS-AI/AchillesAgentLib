Claude Skills

Claude skills in Achilles Agents are Anthropic-style skill bundles that combine a SKILL.md descriptor with optional scripts and reference files. They are executed through an agentic loop session so the model can plan, call tools, and return results using the skill's instructions.

Where They Live

Each Claude skill lives under skills/<domain>/<skill_name>/SKILL.md. The file uses YAML frontmatter followed by Markdown instructions. Optional folders include scripts/, resources/, and assets/. 

---
name: webapp-testing-lite
description: Quick smoke checks for static web pages without external dependencies.
---

# Webapp Testing Lite

## Overview
Use this skill to run a lightweight keyword check against a local static site.

## Steps
1. Use run-script to execute scripts/smoke_check.sh
2. Confirm required keywords exist
3. Return a short PASS/FAIL summary

The frontmatter drives skill discovery (name, description) and the Markdown body provides the operational instructions used by the loop session.

Skill Layout

Claude skills follow the Anthropic bundle format. Only SKILL.md is required; the other folders are optional and loaded on demand. 

my-skill/
├── SKILL.md
├── scripts/
│   └── helper.sh
├── resources/
│   └── checklist.md
└── assets/
    └── template.svg
  • scripts/: Executable helpers invoked via the run-script tool.
  • resources/: Reference files loaded via the get-resource tool.
  • assets/: Output assets (templates, fonts, icons). Not loaded or surfaced directly.

Execution Behaviour

When invoked, Claude skills run inside an agentic loop session. The loop receives the SKILL.md instructions and can call tools as needed.

  • Session: Loop agent session with tool calls and iterative planning.
  • Memory: Uses session memory to continue multi-step tasks.
  • Side effects: Allowed through tool execution (scripts, filesystem reads).

Tooling Integration

ClaudeSkillsSubsystem adapts Anthropic skills by exposing a minimal toolbelt. The loop session decides when to call these tools based on the instructions in SKILL.md.

  • ask-user: Ask the user for missing info or confirmation (exposed via the internal ask-user skill, not the tools folder).
  • read: Read file contents from disk using an absolute file_path.
  • write: Create or overwrite a file with new content.
  • edit: Replace exact text in a file.
  • glob: Find files by glob pattern.
  • grep: Search file contents using regex.
  • bash: Run a shell command.
  • webfetch: Fetch and return text from a URL.
  • run-script: Executes commands in scripts/. Accepts full command strings and resolves paths safely.
  • get-resource: Reads files from resources/. Accepts resources/file or file name.

Assets are intentionally not surfaced. They are available for scripts or downstream tooling but are not injected into LLM context.

Authoring Tips

  • Write explicit steps for when to call scripts or load resources.
  • Keep SKILL.md under 500 lines and move large content into resources.
  • Use concrete inputs/outputs to reduce follow-up questions.
