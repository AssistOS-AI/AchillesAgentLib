Evals Suite

A testing environment designed for LLM‑driven components with non‑deterministic behavior.

Vision

The Evals Suite exists because LLM behavior is not deterministic. Traditional tests assume fixed outputs, but LLM‑based systems can vary between runs even when the inputs are identical. The suite embraces that reality and evaluates reliability at scale rather than single‑run perfection.

A test suite is considered successful when at least 95% of its cases pass. This threshold recognizes natural variance while still enforcing a high standard of correctness.

What It Tests

The suite focuses on agentic workflows and model behaviors that are most sensitive to variability. It runs dozens of cases for each component, compares outputs against expected results, and aggregates pass rates across runs.

  • Loop agentic session behavior and execution quality.
  • Soplang session planning and deterministic execution.
  • Comparative evaluation between Loop and Soplang sessions.
  • Mirror code generation performance and correctness.
  • detectIntents behavior in LLMAgent.
  • Model benchmark comparisons across configured providers.

How It Works

Each suite is driven by JSON test cases that define a prompt, any required context, and an expected outcome. The evaluation harness runs the cases, captures outputs, and scores them against the expected results. A suite passes when at least 95% of cases match the expected behavior.
